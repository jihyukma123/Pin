# 핵심 문제 (Core Problem)

> 이 프로젝트가 해결하려는 단 하나의 페인 포인트.
> 모든 설계/구현 결정은 이 문서를 기준으로 판단한다.

---

## 한 줄 요약

**AI 대화에서 "참조하고 싶은 응답"이 후속 질문을 던지는 순간 스크롤에 묻혀버린다.**
사용자는 그 응답을 화면에 **고정(pin)** 한 채로, 같은 세션에서 자유롭게 후속 대화를 이어가고 싶다.

---

## 상세 시나리오

1. 사용자가 학습 목적으로 AI(Claude Code, Codex 등)에게 질문을 던진다.
2. AI가 길이가 긴, 정보 밀도 높은 응답을 돌려준다.
3. 사용자가 그 응답을 읽다 보면 자연스럽게 후속 질문이 생긴다.
4. **같은 세션 안에서** 후속 질문을 이어가고 싶다 — 맥락을 유지하기 위해.
5. 그러나 후속 질문을 입력하는 순간, 원본 응답이 위로 밀려 화면에서 사라진다.
6. 결국 "원본 응답 ↔ 후속 답변" 사이를 스크롤로 오가야 한다.
7. 맥락이 끊기고, 인지 부하가 커지고, 학습 흐름이 깨진다.

## 문제의 본질

대화의 **흐름**은 선형(linear)이지만,
사용자가 다루고 싶은 **참조 대상**은 비선형(non-linear, 고정)이다.

기존 채팅 UI는 이 둘을 분리하지 않고 같은 스크롤 축에 묶어둔다.
→ 참조하고 싶은 응답은 시간이 지날수록 멀어지고, 결국 사라진다.

## 사용자가 원하는 것

- 특정 응답(또는 응답의 일부)을 **시각적으로 고정**할 수 있어야 한다.
- 고정된 동안에도 **같은 세션**에서 대화를 자연스럽게 이어갈 수 있어야 한다.
- 고정 대상을 **여러 개** 둘 수 있으면 더 좋다 (학습 중에는 참조점이 누적된다).
- 고정을 **해제**하는 것도 쉬워야 한다.

## 비-목표 (Non-goals)

- 새로운 LLM/모델을 만드는 것이 아니다.
- 기존 AI 도구를 대체하는 것이 아니다.
- 대화 히스토리 검색/아카이브 도구가 아니다.

핵심은 오직 하나 — **"참조 응답을 화면에 붙들어 두는 경험"**.
